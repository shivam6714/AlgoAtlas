# Selection Sort

Selection Sort is a simple comparison-based sorting algorithm. It divides the input array into two parts: a sorted subarray that is built up from left to right at the front of the array, and an unsorted subarray occupying the remainder of the array.

---

## Introduction

Selection Sort repeatedly searches for the smallest (or largest) element from the unsorted segment of the array and swaps it with the first element of the unsorted segment.

---

## How It Works

1. Start with the first element of the array as the minimum.
2. Compare this minimum with each element in the remaining unsorted portion.
3. If a smaller element is encountered, update the index of the minimum element.
4. After scanning the unsorted portion, swap the minimum element with the element at the beginning of the unsorted portion.
5. Advance the boundary between the sorted and unsorted portions by one element and repeat until the array is sorted.

---

## Algorithm

```cpp
void selectionSort(vector<int>& arr) {
    int n = arr.size();

    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        if (minIndex != i) {
            swap(arr[i], arr[minIndex]);
        }
    }
}
```

---

## Example

Consider the array `[64, 25, 12, 22, 11]`:

- **Pass 1:** Find min in `[64, 25, 12, 22, 11]` (min is `11` at index 4). Swap with index 0. Array becomes `[11, 25, 12, 22, 64]`.
- **Pass 2:** Find min in `[25, 12, 22, 64]` (min is `12` at index 2). Swap with index 1. Array becomes `[11, 12, 25, 22, 64]`.
- **Pass 3:** Find min in `[25, 22, 64]` (min is `22` at index 3). Swap with index 2. Array becomes `[11, 12, 22, 25, 64]`.
- **Pass 4:** Find min in `[25, 64]` (min is `25` at index 3). No swap needed.
- **Pass 5:** Array is fully sorted: `[11, 12, 22, 25, 64]`.

---

## Advantages

- Simple and straightforward to implement.
- Performs well on small datasets.
- In-place algorithm; requires only O(1) auxiliary memory.
- Minimizes memory writes (O(n) total swaps).

---

## Disadvantages

- Time complexity is O(n²) regardless of initial array ordering.
- Inefficient on large datasets.
- Not a stable sort by default (relative order of identical keys may change).

---

## Complexity Explanation

| Case | Time Complexity |
|------|-----------------|
| Best Case | O(n²) |
| Average Case | O(n²) |
| Worst Case | O(n²) |

- **Time Complexity:** The algorithm employs two nested loops. The outer loop runs n - 1 times, and the inner loop performs n - i - 1 comparisons per iteration. The total number of comparisons is n × (n - 1) / 2, which simplifies to O(n²) in all cases.
- **Space Complexity:** **O(1)** auxiliary space since swapping is performed in-place.

---

## When to Use

- When writing to memory is an expensive operation (since Selection Sort makes at most n - 1 swaps).
- For small datasets where the overhead of complex O(n log n) algorithms is unnecessary.
- In memory-constrained environments requiring strict O(1) space.
