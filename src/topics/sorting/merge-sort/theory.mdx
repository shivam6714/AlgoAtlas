# Merge Sort

Merge Sort is an efficient, comparison-based, divide-and-conquer sorting algorithm. It divides the array into two halves, recursively sorts them, and then merges the two sorted halves.

---

## Introduction

Merge Sort guarantees O(n log n) performance regardless of the initial order of elements by breaking the array into single-element subarrays and merging them in sorted order.

---

## How It Works

1. Divide: Find the middle point of the array and divide it into left and right halves.
2. Conquer: Recursively sort the left half and right half.
3. Combine: Merge the two sorted halves into a single sorted array.

---

## Step-by-Step Explanation

- The base case for recursion is an array with 1 or 0 elements, which is inherently sorted.
- For arrays with 2 or more elements, calculate mid = left + (right - left) / 2.
- Make recursive calls mergeSort(arr, left, mid) and mergeSort(arr, mid + 1, right).
- Use a temporary array in the merge step to compare elements from the left and right subarrays, copying the smaller element first into the temporary array.
- Copy the sorted elements back from the temporary array into the original array segment.

---

## Algorithm

```cpp
#include <vector>
using namespace std;

void merge(vector<int>& arr, int left, int mid, int right) {
    vector<int> temp;
    int i = left;
    int j = mid + 1;

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp.push_back(arr[i++]);
        } else {
            temp.push_back(arr[j++]);
        }
    }

    while (i <= mid) temp.push_back(arr[i++]);
    while (j <= right) temp.push_back(arr[j++]);

    for (int k = 0; k < temp.size(); k++) {
        arr[left + k] = temp[k];
    }
}

void mergeSort(vector<int>& arr, int left, int right) {
    if (left >= right) return;

    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}
```

## Dry Run

Consider the array `[38, 27, 43, 3, 9, 82, 10]`:

- **Division Phase:**
  - Split into `[38, 27, 43, 3]` and `[9, 82, 10]`
  - Split into `[38, 27]` and `[43, 3]` | `[9, 82]` and `[10]`
  - Base cases reached: `[38]`, `[27]`, `[43]`, `[3]`, `[9]`, `[82]`, `[10]`
- **Merge Phase:**
  - Merge `[38]` and `[27]` → `[27, 38]`
  - Merge `[43]` and `[3]` → `[3, 43]`
  - Merge `[27, 38]` and `[3, 43]` → `[3, 27, 38, 43]`
  - Merge `[9]` and `[82]` → `[9, 82]`
  - Merge `[9, 82]` and `[10]` → `[9, 10, 82]`
  - Merge `[3, 27, 38, 43]` and `[9, 10, 82]` → `[3, 9, 10, 27, 38, 43, 82]`

## Advantages

- Guaranteed O(n log n) time complexity in worst, average, and best cases.
- Stable sort algorithm.
- Excellent choice for sorting linked lists with O(1) extra space.
- Parallelizable since division of subarrays can be handled independently.

---

## Disadvantages

- Requires O(n) additional space for temporary arrays during the merge phase.
- Slower for small arrays compared to Insertion Sort or Quick Sort due to recursive overhead and memory allocations.

---

## Complexity Explanation

| Case | Time Complexity |
|------|-----------------|
| Best Case | O(n log n) |
| Average Case | O(n log n) |
| Worst Case | O(n log n) |

- **Time Complexity:** Dividing the array takes log n levels of recursion. At each level, merging elements across subarrays takes total O(n) time. Total time complexity is O(n log n) in all cases.
- **Space Complexity:** **O(n)** extra memory is required to hold temporary arrays during the merge operation.

---

## When to Use

- When guaranteed performance of O(n log n) is required.
- When stability is required for sorting identical elements.
- When sorting external data or large linked lists.
