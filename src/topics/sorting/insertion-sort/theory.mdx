# Insertion Sort

Insertion Sort is a simple comparison-based sorting algorithm that builds the sorted array one item at a time. It works similarly to the way playing cards are sorted in your hand.

---

## Introduction

Insertion Sort takes elements one by one from an unsorted array and inserts them into their correct position in a previously sorted portion of the array.

---

## How It Works

1. Assume the first element is already sorted.
2. Pick the next element (key) from the unsorted portion.
3. Compare the key with elements in the sorted portion from right to left.
4. Shift all elements greater than the key one position to the right.
5. Insert the key into its correct sorted position.
6. Repeat for all remaining elements until the array is fully sorted.

---

## Step-by-Step Explanation

- The array is logically split into a sorted sub-array on the left and an unsorted sub-array on the right.
- Initially, the sorted sub-array contains only the element at index 0.
- For index i from 1 to n - 1, we store arr[i] as the key.
- We shift all elements in arr[0 ... i-1] that are greater than key to position j + 1.
- We place key at position j + 1 once a smaller or equal element is found.

---

## Algorithm

```cpp
void insertionSort(vector<int>& arr) {
    int n = arr.size();

    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;

        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }

        arr[j + 1] = key;
    }
}
```

---

## Dry Run

Consider sorting the array `[12, 11, 13, 5, 6]`:

- **Initial State:** Sorted `[12]`, Unsorted `[11, 13, 5, 6]`
- **Pass 1 (i = 1, key = 11):**
  - Compare 11 with 12. Since 12 > 11, shift 12 to right.
  - Insert 11 at index 0.
  - Array becomes `[11, 12, 13, 5, 6]`.
- **Pass 2 (i = 2, key = 13):**
  - Compare 13 with 12. 12 < 13, no shift.
  - Insert 13 at index 2.
  - Array becomes `[11, 12, 13, 5, 6]`.
- **Pass 3 (i = 3, key = 5):**
  - Compare 5 with 13, 12, 11. Shift 13, 12, 11 right.
  - Insert 5 at index 0.
  - Array becomes `[5, 11, 12, 13, 6]`.
- **Pass 4 (i = 4, key = 6):**
  - Compare 6 with 13, 12, 11. Shift 13, 12, 11 right. 5 < 6, stop.
  - Insert 6 at index 1.
  - Final Array: `[5, 6, 11, 12, 13]`.

---

## Advantages

- Simple implementation and easy to understand.
- Efficient for small datasets.
- Adaptive: Very fast for nearly sorted arrays (O(n) time).
- Stable: Preserves the relative order of equal keys.
- In-place sorting: Uses O(1) extra space.

---

## Disadvantages

- Inefficient for large datasets with average and worst-case O(n²) time complexity.
- Requires high number of element shifts in reverse-sorted arrays.

---

## Complexity Explanation

| Case | Time Complexity |
|------|-----------------|
| Best Case | O(n) |
| Average Case | O(n²) |
| Worst Case | O(n²) |

- **Best Case O(n):** Occurs when the input array is already sorted. The inner loop condition fails immediately on every pass.
- **Worst Case O(n²):** Occurs when the input array is sorted in reverse order. Every element must be compared and shifted across the entire sorted sub-array.
- **Space Complexity O(1):** Sorting is performed in-place with no additional data structures.

---

## When to Use

- When the input array is small.
- When the data is already nearly sorted.
- When an online sorting algorithm is needed (sorting data as it arrives).
