# Quick Sort

Quick Sort is an efficient, comparison-based, divide-and-conquer sorting algorithm. It works by selecting a 'pivot' element from the array and partitioning the other elements into two sub-arrays according to whether they are less than or greater than the pivot.

---

## Introduction

Quick Sort is one of the most widely used sorting algorithms in practice because of its cache efficiency, low memory footprint, and fast average-case performance.

---

## How It Works

1. Pick a pivot element from the array (e.g., the last element).
2. Partitioning: Rearrange the array so that all elements smaller than the pivot come before it, and all elements greater than the pivot come after it.
3. Recursively apply Quick Sort to the sub-array of smaller elements and the sub-array of larger elements.

---

## Step-by-Step Explanation

- Select a pivot strategy (commonly last element arr[high]).
- Initialize an index i pointing to the boundary of smaller elements.
- Loop j from low to high - 1: if arr[j] < pivot, increment i and swap arr[i] with arr[j].
- After the loop, swap arr[i + 1] with arr[high] to place the pivot in its correct sorted position.
- Return i + 1 as the pivot index.
- Recursively call quickSort on low to pivotIndex - 1 and pivotIndex + 1 to high.

---

## Algorithm

```cpp
#include <vector>
#include <algorithm>
using namespace std;

int partition(vector<int>& arr, int low, int high) {
    int pivot = arr[high];
    int i = low - 1;

    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[high]);
    return i + 1;
}

void quickSort(vector<int>& arr, int low, int high) {
    if (low < high) {
        int pivotIndex = partition(arr, low, high);
        quickSort(arr, low, pivotIndex - 1);
        quickSort(arr, pivotIndex + 1, high);
    }
}
```

## Dry Run

Consider the array `[10, 80, 30, 90, 40, 50, 70]` with last element `70` as pivot:

- **Partition Phase (pivot = 70):**
  - Compare 10 < 70 → Swap 10 with itself (i = 0).
  - Compare 80 > 70 → No swap.
  - Compare 30 < 70 → Swap 30 with 80 (i = 1).
  - Compare 90 > 70 → No swap.
  - Compare 40 < 70 → Swap 40 with 80 (i = 2).
  - Compare 50 < 70 → Swap 50 with 90 (i = 3).
  - Swap pivot 70 with 80 (i + 1 = 4).
  - Array after partition: `[10, 30, 40, 50, 70, 90, 80]`. Pivot index = 4.
- **Recursive Phase:**
  - Recursively sort left sub-array `[10, 30, 40, 50]`.
  - Recursively sort right sub-array `[90, 80]`.
  - Final sorted array: `[10, 30, 40, 50, 70, 80, 90]`.

---

## Advantages

- Highly efficient on average with O(n log n) running time.
- In-place sorting algorithm requiring only O(log n) auxiliary stack memory.
- Low constant factors and excellent cache performance.

---

## Disadvantages

- Worst-case time complexity is O(n²) when bad pivots are selected (e.g., sorted array with last element as pivot).
- Unstable sort algorithm (does not preserve relative order of equal elements).

---

## Complexity Explanation

| Case | Time Complexity |
|------|-----------------|
| Best Case | O(n log n) |
| Average Case | O(n log n) |
| Worst Case | O(n²) |

- **Best/Average Case O(n log n):** Occurs when the pivot splits the array into roughly equal halves at each step.
- **Worst Case O(n²):** Occurs when the pivot chosen is consistently the smallest or largest element, producing unbalanced partitions.
- **Space Complexity O(log n):** Auxiliary space used by the call stack for recursion in balanced partitions.

---

## When to Use

- When fast average-case sorting performance is desired.
- When space efficiency is critical and O(n) extra space of Merge Sort cannot be afforded.
- In general-purpose systems programming (e.g., C++ std::sort uses Introsort, a hybrid of Quick Sort, Heap Sort, and Insertion Sort).
